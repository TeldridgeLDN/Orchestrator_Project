### Task 4: A/B Test Setup

**Test Variants:**

**Variant A (Control)**: No examples section  
**Variant B (Dedicated Section)**: Examples section after packages  
**Variant C (Inline Links)**: "See Example" buttons in package cards  

**Setup:**

1. **Create Test Configuration**
   - Use Plausible custom events or manual tracking
   - Track:
     - Variant shown (A, B, or C)
     - Examples section views
     - "See Example" clicks
     - Package purchase clicks (per variant)

2. **Implement Variant Logic**
   - Use URL parameter: `?variant=a` / `?variant=b` / `?variant=c`
   - OR: Random assignment on page load (50/25/25 split)
   - Store variant in localStorage to maintain consistency

3. **Deploy Test Pages**
   - `/validate/?variant=a` (Control)
   - `/validate/?variant=b` (Section)
   - `/validate/?variant=c` (Inline)

4. **Set Up Analytics Dashboard**
   - Create Plausible goals:
     - `example_section_view`
     - `example_page_view`
     - `package_click_from_example`
     - `package_purchase_intent`

**Estimated Time**: 3-4 hours

## Success Criteria

### Example Page Quality Checklist

Each example page must:
- ✅ Load in <2 seconds (optimized images)
- ✅ Be fully mobile-responsive
- ✅ Have WCAG AA color contrast
- ✅ Include validation labels (time + results)
- ✅ Look polished (no placeholder text, all images loaded)
- ✅ Match industry aesthetic (plant shop = earthy, coffee = minimalist)

### A/B Test Success Metrics

**Primary Metric**: Package purchase conversion rate
- Target: >15% lift in Variant B or C vs. Control

**Secondary Metrics**:
- Example page views (engagement)
- Time on examples (indicates interest)
- Scroll depth on validation pages

**Run Duration**: 2 weeks or 500+ visitors (whichever comes first)
